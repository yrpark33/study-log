# 플래시 어트리뷰트가 리다이렉트 두 번 거치면 사라지는 문제

## 1. 문제 제기: "에러 메시지가 간헐적으로 출력되지 않음"

`@ControllerAdvice`에서 예외 발생 시 `rttr.addFlashAttribute("errorMsg", msg)`로 메시지를 세팅하고 `redirect:/`로 보냈는데, alert가 간헐적으로 안 뜨는 문제가 발생했습니다.

## 2. 원인분석

### ① 플래시 어트리뷰트의 생명주기

플래시 어트리뷰트는 리다이렉트를 **한 번** 거치는 동안만 살아있습니다. 리다이렉트 응답을 받은 다음 GET 요청 시 세션에서 꺼내서 모델에 담고, 그 즉시 세션에서 삭제됩니다.

### ② 리다이렉트가 두 번 발생

```java
// ControllerAdvice
return "redirect:/"; // 1차 리다이렉트

// HomeController
@GetMapping("/")
public String home() {
    return "redirect:/board/list"; // 2차 리다이렉트
}
```

`redirect:/`에서 `redirect:/board/list`로 한 번 더 리다이렉트되면서 플래시 어트리뷰트가 소비되어 사라졌습니다.

## 3. 해결

`ControllerAdvice`에서 목적지를 직접 지정했습니다.

```java
@ExceptionHandler(ApplicationException.class)
public String handle(ApplicationException ex, RedirectAttributes rttr) {
    rttr.addFlashAttribute("errorMsg", msg);
    return "redirect:/board/list"; // 직접 목적지 지정
}
```

## 4. 결론 및 회고

- **플래시 어트리뷰트는 리다이렉트 한 번만 살아있다**: 리다이렉트 체인이 있으면 중간에 소비되어 사라집니다.
- **리다이렉트 체인 주의**: `redirect:/`처럼 중간 경유지로 보내면 플래시 어트리뷰트가 소실될 수 있으므로 최종 목적지로 직접 보내는 것이 안전합니다.
